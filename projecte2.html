PROJECT 2: DB SECURITY & SQLi

1. Objectives of the Project

This project focuses on the secure logical design of relational databases and the prevention of SQL injection (SQLi) The goal is to ensure data integrity through normalization and the use of parameterized queries

2. Vulnerability Analysis (PoC)

It was demonstrated how legacy code without sanitization allowed administrative access by bypassing authentication.

Vector d'Atac (Payload):

input_login_field

Vulnerable Code (PHP):

vulnerable_login.php
$sql = "SELECT * FROM users WHERE user = '" . $user . "'";
// This code concatenates strings directly -> UNSAFE

Solution Implemented (Prepared Statements):

secure_login.php
$stmt = $pdo->prepare('SELECT * FROM users WHERE user = :user');
$stmt->execute(['user' => $input_user]);
// Using parameters prevents SQL injection.

3. Conclusions

Database design should follow normal forms (1NF, 2NF, 3NF) to avoid redundancy, but security lies at the application layer: user input should never be trusted.
